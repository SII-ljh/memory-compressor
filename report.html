QCPC

Query-Conditioned Perceiver Compressor — 基于 Perceiver IO 的长文本压缩方案，将长度 N 的文本压缩为 M 个 Memory Token 作为 Soft Prompt 送入冻结 LLM，计算量从 O(N²) 降至 O(M·N)

模型架构

QCPC 采用五阶段流水线：长文本经冻结的 Embedding 层获得词向量后，由 Perceiver IO 的 Read（交叉注意力）和 Process（自注意力）阶段压缩为 M 个 Memory Token，最终拼接提示词送入冻结的 Qwen3 Decoder 进行生成。

Decoupled RoPE 机制

在 Read 阶段的 Cross-Attention 中，Q 和 K 各拆分为内容通道与位置通道：

  • KV 侧（长文本）：内容通道不施加 RoPE；位置通道独立投影后施加 RoPE，编码真实绝对位置。
  • Query 侧（Latent）：内容通道不施加 RoPE；位置通道使用可学习 Slot Position Embedding，每个 slot 自由学习区域偏好。

注意力得分 = 内容匹配 + 位置偏好的叠加，内容与位置互不干扰。

训练数据

Stage 1 — 文本补全预训练数据

大规模无监督长文本语料，JSONL 格式（{"text": "...", "id": "..."}）。每条文本被分词后切分为固定长度的 segment，再拆分为前缀（送入压缩器）和续写部分（作为生成目标）。

  • Stage 1a（热身）：context=512, continuation=128
  • Stage 1b（多 chunk）：chunk_len=512, 2~8 chunks 动态拼接, continuation=128

数据路径：data/stage1/train.jsonl、data/stage1/warmup_train.jsonl

Stage 2 — QA 指令微调数据

(Context, Question, Answer) 三元组，JSON 格式。Context 最长 4096 token，Prompt 最长 128 token，Answer 最长 256 token。

数据路径：data/stage2/train.json、data/stage2/eval.json

训练方式

Stage 1a — 短窗口热身

目标：学习基础的文本压缩能力

  • Context 512, Continuation 128
  • Prompt Bias 冻结（输出恒为 0）
  • LR: 1e-4, Batch: 8, Grad Accum: 4
  • Warmup tokens: 500M
  • Loss: 续写部分 Next-Token CE

Stage 1b — 多 Chunk 长文本

目标：扩展到长上下文压缩

  • Context 最长 4096 (2~8 chunks × 512)
  • 逐 chunk 输入，Memory 跨 chunk 累积
  • Prompt Bias 仍冻结
  • 从 1a checkpoint 继续训练
  • Loss: 续写部分 Next-Token CE

Stage 2 — QA 指令微调

目标：激活 Prompt-Guided 定向压缩

  • 解冻 QueryMapper + α
  • 零初始化保证平滑过渡
  • LR: 2e-5, Batch: 4
  • Context→压缩，Question→偏置
  • Loss: 仅 Answer 部分 CE
零初始化平滑过渡：QueryMapper 第二层权重/偏置和 α 均初始化为 0，保证 Stage 2 开始时模型行为与 Stage 1 完全一致，Prompt 信号从零渐进注入（类似 ReZero / ControlNet 范式）。

评估方式

Stage 1 评估

在 held-out 文本上计算续写的 Loss 和 Perplexity (PPL)，衡量 Memory Token 的通用语义保持能力。

  • Full benchmark: context=4096, continuation=256, 500 samples
  • 分别在 1a（短窗口）和 1b（多 chunk）设置下评估
Stage 2 评估

在 QA eval 集上评估生成质量，指标包括：

  • Loss / PPL：Answer 部分的交叉熵和困惑度
  • ROUGE-L：最长公共子序列 F-score
  • F1：Token 级别的 F1 分数
  • EM：Exact Match 精确匹配率

同时设置 Upper Bound（LLM 直接接收完整上下文）和 Lower Bound（使用压缩后的 Memory Token）用于对比。评估使用 1,999 样本。

关键发现：
  • Upper Bound 下 Qwen3-4B 可达 F1=0.885、EM=0.827，说明 decoder 本身具备较强 QA 能力。
  • 当前 QCPC 的压缩 Memory 与 Upper Bound 存在较大差距（F1 下降 53-59%），表明压缩后的信息损失仍然显著。
  • QCPC 训练后的 ROUGE-L / F1 / EM 极低（~0.01），Stage 2 微调尚未有效将压缩信息转化为可用的 QA 答案生成能力，后续需进一步改进。
  • PPL 方面，QCPC 在 Stage 2 的 loss 随模型增大持续下降（11.3 → 7.5 → 5.6），说明语言建模层面的压缩质量在提升。

总结

  • 架构验证：五阶段流水线成功实现 O(M·N) 复杂度的长文本压缩，各组件（Decoupled RoPE、零初始化 Prompt Bias）均按预期工作。
  • Stage 1 预训练：Memory Token 有效捕获文本语义，PPL 随模型规模和 Memory 数量提升而改善，最优配置 Qwen3-4B + M=128 达到 PPL=12.07。
  • Stage 2 瓶颈：QA 微调阶段生成质量（ROUGE-L、F1、EM）远低于预期，压缩 Memory 到可精确回答的转化仍是核心挑战。Upper/Lower Bound 差距为后续优化提供了明确的改进空间。
  • Scaling 规律：更大的 decoder 模型在 Stage 1 和 Stage 2 的所有指标上均表现更优，验证了方案对不同规模模型的适用性。
